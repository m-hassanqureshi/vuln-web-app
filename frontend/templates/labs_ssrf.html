Lab Briefing

SSRF occurs when a web application fetches a remote resource (like an API endpoint or image) without validating the user-supplied URL. An attacker can craft a request pointing to internal resources (such as loopback services or metadata endpoints like 169.254.169.254) that the web server can access but are not exposed to the public internet.

Objective:
  1. Request an internal endpoint in Vulnerable Mode (e.g., fetch http://localhost:3001/admin).
  2. Verify that the status changes to EXPLOITED.
  3. Toggle to Secure Mode and check that loopback/private addresses are blocked. Once verified, the lab becomes SOLVED!
