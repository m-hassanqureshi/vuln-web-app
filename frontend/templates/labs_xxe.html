Lab Briefing

XXE Injection occurs when an XML parser processes a user-supplied XML document that contains references to external entities. If the parser is configured to resolve external entities and support Document Type Definitions (DTDs), an attacker can declare an entity that references local files or internal endpoints.

Objective:
  1. Copy the sample payload from below. Adjust the path to a file on your server (e.g., file:///c:/windows/win.ini for Windows or file:///etc/passwd for Unix/Linux).
  2. Submit the payload in Vulnerable Mode and check if the file contents are parsed and displayed on screen.
  3. Verify that the status changes to EXPLOITED.
  4. Toggle to Secure Mode and resubmit the payload. Verify that the parser rejects or ignores entity resolution. Once verified, the lab becomes SOLVED!

Sample XML Payload:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE user [
  <!ENTITY xxe SYSTEM "file:///c:/windows/win.ini">
]>
<user>
  <name>&xxe;</name>
  <email>student@fccu.edu.pk</email>
</user>
